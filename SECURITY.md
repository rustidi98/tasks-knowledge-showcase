# Security policy

This is a **case-study / showcase repository**. It contains sanitized architecture
notes and representative code extracts — no production source, no secrets, no
credentials, and no patient or customer data. The live product it describes is a
private, commercial healthcare application; its source is not published here.

## Reporting a concern

If you believe something in this repository was published in error — a secret, a
credential, an internal hostname, a real client name, or anything that looks like
protected data slipped through sanitization — please report it privately rather
than opening a public issue:

**rust.idi98@gmail.com**

I'll acknowledge within a few days and remove or rewrite the offending content
promptly. Thank you for the responsible disclosure.

## What is deliberately not here

- Production credentials, API keys, tokens, or connection strings.
- Real client, patient, host, or infrastructure identifiers.
- The private application source and its live integrations.
